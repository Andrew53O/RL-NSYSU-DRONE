# Yuan et al., 2021 — AUV Obstacle Avoidance Planning Based on Deep Reinforcement Learning

**Citation / Goal**
- Yuan et al., 2021. "AUV Obstacle Avoidance Planning Based on Deep Reinforcement Learning" (Journal of Marine Science and Engineering).
- Goal: use processed active-sonar information (not raw images) to train RL agents for obstacle avoidance in underwater vehicles (AUVs). Key takeaway: compress sonar beams into compact features to stabilize and speed learning.

**Sensors / Context**
- Active sonar array (many beams) used to perceive nearby obstacles.
- Experiments in static, mixed static/dynamic, and dynamic obstacle scenarios.

**Core idea**
- Convert raw sonar beams into a low-dimensional set of distance/risk features (sector aggregation, clipped ranges, normalized values).
- Train RL (variants used in the paper) on the processed state rather than raw high-dimensional perception to improve sample efficiency and robustness.

**Observation / State design**
- Combine: processed sonar sector distances (or aggregated beam features), vehicle pose/velocity, and target vector.
- Normalization: scale position and range features by known workspace or sensor maxima (e.g., position by boundary size, sonar by max range) to keep neural net inputs well-conditioned.

**Range → Risk conversion (recommended)**
- Use a clipped linear mapping from range r to a bounded risk score:

  risk = clip((caution_distance - r) / caution_distance, 0, 1)

  - `caution_distance` is a design constant (e.g., 1.5 m). Risk ∈ [0,1].

**Reward shaping**
- Combine progress-to-goal, distance penalty, collision penalty, and obstacle proximity penalties (mean/max sector risk).
- Typical components: progress_reward (previous_dist - current_dist), collision terminal penalty, mean-risk penalty, max-risk penalty. Shaping helps learning by providing dense feedback rather than relying on sparse crash signals.

**Action space**
- Paper experiments vary (discrete/continuous). Design takeaway: match the action abstraction to the simulator/controller (e.g., continuous velocity commands if the simulator accepts velocity setpoints).

**Metrics / experiments**
- Report success rate, collision rate, sample efficiency. Processed sonar state outperforms raw high-dimensional inputs in learning speed and stability.

**Implementation notes for porting to drone Task D**
- Aggregate front-facing beams into logical sectors: left, center, right, up, down.
- Normalize sectors by sonar max range (e.g., 10 m) and positions by workspace bounds (e.g., x/y scale = 8 m, z scale = 5 m).
- Use continuous PPO for smooth velocity outputs if simulator provides a low-level velocity controller.
- Add short-term memory (previous readings, trends) if sonar is partially observable — see Mane et al.

**Report language snippets**
- "Following Yuan et al., we convert active-sonar beams into a compact sector-based distance representation and feed processed sonar features into PPO rather than raw signals. This reduces input dimensionality, stabilizes learning, and enables effective reward shaping for obstacle-aware navigation."

*Notes for implementers:* copy the risk mapping and sectorization strategy directly into observation preprocessing code. Use mean/max risk penalties in reward shaping to punish both overall clutter and single close obstacles.
